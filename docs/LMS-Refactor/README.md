## How to read these wireframes

Each file contains:

1. **Header block** — platform, autonomy level, authoring effort, mobile suitability
2. **ASCII wireframe(s)** — structural layout using box-drawing characters
3. **Structure breakdown table** — zone-by-zone description of UI regions
4. **Component reference** — item types, states, and indicators
5. **Pedagogical model** — the learning theory embedded in the design
6. **Key design constraints** — practical rules for implementation and content authoring

The wireframes are intentionally schematic. They document layout architecture, navigation logic, and content organisation — not visual design. Colour, typography, and branding are applied at the theme layer during implementation.

## SKIP implementation notes

The CAPE Framework (Capacity, Adoption, Pedagogical Enhancement, Ecosystem Integration) governs graduated institutional deployment across OECS member states. Layout selection should be mapped to institutional tier:

| CAPE tier | Recommended layout | Rationale |
|---|---|---|
| Early adopter | Layout 01 or Layout 02 | Low authoring burden; clear navigation for new online learners |
| Developing | Layout 02 or Layout 03 | Increased autonomy; visual catalogue for broader programme range |
| Advanced | Layout 04 or Layout 05 | Full instructional design capacity; SCORM integration; adaptive delivery |

Layout 05 (Adaptive Learning Path) should be treated as a medium-term target. The data infrastructure, assessment calibration, and content variant development it requires exceed the capacity of the initial SKIP deployment phase.

The LearnBoard-centred platform pathway supports multiple layout paradigms within a single architecture, enabling differentiated deployment across member states without sacrificing interoperability.
